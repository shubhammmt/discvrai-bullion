## Slide 3: Connecting the Dots — Pre-Sales to Post-Sales

**Headline**: Same Student, Same Platform: One Identity From Lead to Completion

**Flow**:
- **Pre-sales**: Prospect → Lead ID → qualification → enrichment → enrollment.
- **Post-sales**: Same Lead ID / student ID → induction (Day 0–7) → session-based touchpoints (Day 8–90) → retention score → fees reminders → query handling → post-course / next-step recommendations (same interface) → (later) at-risk list, Lost Customer survey.

**Key message**:
We don't hand off to a different system. The agentic layer that qualifies and enriches also runs onboarding, triggers, surveys, fees, and support. Internal heads see one funnel: campaign → lead → enrollment → retention → outcome.

**Visual**: Single horizontal journey: Lead ID → Enrollment → Onboarding → Retention → Fees → Predictive / Recommendation; "One platform, one identity" callout.

## Slide 5: One Interface for the User — Multiple Agents Behind It

**Headline**: Same Conversation, Same Thread: WhatsApp (or Embedded Chat) Is the Single User Interface

**Key message**:
The student or prospect never switches apps. One conversational interface — WhatsApp or a chat widget on your existing site/ProConnect — handles the full journey. Behind the scenes we run multiple agents (career counsellor, onboarding, fees, query, recommendation); to the user it's one continuous conversation.

| User sees | Behind the scene (agents) |
|-----------|--------------------------|
| One WhatsApp thread (or one chat) | A1 Qualification → A2 Enrichment → A5 Onboarding → A6 Fees → A7 Query → A9 Recommendation |
| Same place for enquiry, induction, reminders, questions, "What's next?" | Right agent responds by context (pre-enrollment vs post-enrollment, trigger, intent) |

Where this interface can sit: WhatsApp Business; embedded chat on Aptech site; in-app chat inside ProConnect/Appra (when integrated). One interface, existing applications — we plug in.

**Visual**: One phone/chat window with a single thread; callout "One interface. Multiple agents. One conversation."

## Slide 5c: Post-Course Recommendations — Highlight

**What we enable**:
In the same conversational interface (WhatsApp or portal), we surface post-course / next-step recommendations: next course, advanced module, workshop, or certification — based on current course, progress, engagement score, and completion.

| When | What student sees (same WhatsApp/chat) |

| Mid-course (e.g. after session 20) or near completion | "Based on your progress in [Course], you might like: [Next Course / Workshop]. Interested?" |

**Why it matters**: Upsell and cross-sell without a new touchpoint; the student already talks to us in one place. Counsellors see "upgrade-ready" list (I6); HQ sees recommendation revenue and uptake.

## Slide 7: Trigger Framework — The Heart of Onboarding

**Headline**: Org-Level Triggers Drive When We Act (No Guesswork)

**Examples**:
- 2 consecutive absences → "We missed you" WhatsApp (+ optional voice); student appears in counsellor "attention" queue.
- Every 4th session → Short satisfaction survey (e.g. 1–5 + "Any block?").
- Faculty / module change → "New faculty/module" message + optional sentiment question.
- Fee overdue 7 days → Escalation message + optional "assign to counsellor."

**We build**: A configurable trigger engine (events from Aptrack 2.0 / ProConnect). Aptech defines the rules; we run the sub-routines (message, survey, call, escalate).

**Visual**: Flow: Event (e.g. "2 absences") → Trigger fires → Sub-routine (WhatsApp + queue) → Counsellor sees student.

## Slide 12: Why This Fits — Same Philosophy as Pre-Sales

**Headline**: Business Transformation, Not Just IT — Deep Integration, Metrics, Extensibility

- **Same platform**: Pre-sales and post-sales on one agentic layer; one Lead ID/student ID.
- **Augment, not replace**: Counsellors get queues and context; AI handles reach-out and reminders.
- **Metric-driven**: Retention score, collection rate, at-risk count, Lost Customer reasons — all visible to internal heads.
- **Extensible**: New triggers, new channels, new agents can plug in (API-first, event-driven).
- **Aptech-owned outcomes**: We enable; Aptech owns process and decisions.
